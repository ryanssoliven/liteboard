Conclusion

Did Django and Materialize perform to your expectations?

Django exceeded my expectations. One particular expectation that I had was that the code would be difficult to understand and implement. In reality, the code was actually very simple to read and write, due to Django utilizing Python. As well, Django was much more powerful than I expected. As detailed in the tutorial, entire databases could be constructed in a simple file of no more than 10 lines of code.

Materialize exceeded expectations, I thought it would be very limited with what you can do with it, but with added javascript it has a lot of potential

Were Django and Materialize difficult to install or configure?

Django was really easy to install. Since the server hosting this website is built on ubuntu, Python and Django were simple apt-get and pip installs respectively. Creating the web application architecture was simple as well, by running a handful of commands our back end architecture was set up and ready to configure. Configuration was difficult at times as Django is very strict with how servers are to be set up as to enforce good website design principles, but the tradeoff is that when you understand Django simple commands yield powerful results. One hiccup we could not resolve was using our server to host static files as Django has difficulties doing so once the server was pushed from a development server to the live server. We worked around this by embedding css into style tags at the tops of our headers, and hosting images on a third party website.

Materialize was very simple to install and configure. You just need to link to Materialize stylesheet, import JQuery, and import the Materialize JavaScript

Was it easy to create the page(s) with Django and Materialize?

There are actually quite a few steps to set up before you can use a page you created in Django. For example, a new html page has to have a view associated with it, which if it has a seperate address associated with it, needs a url to be configured and allowed by Django. It was initially difficult to understand how to create pages, but became easier as I understood Django.

It wasn't all to diffult to add Materialize components onto the Django website since it's just HTML. Understanding the jinja was the only part that really needed to be used intertwine with the materialize code. There were some limitations where I could not figure out how to access the django forms and database with Materialize

Was the learning curve steep compared with regular HTML / CSS / Javascript / PHP?

For Django, it was quite steep. To learn it I had to follow along a YouTube tutorial split over 12 parts that detailed a websites creation from start to finish. It was the first time I’ve ever used a backend that enforced a model-view-template architecture, which was difficult to understand prior to learning it.

The learning curve was probably simpler compared to learning HTML/CSS/Javascript/PHP from scratch. Materialize had simple to navigate documentation. Once you understand html and how to initilize using JQuery than everything goes pretty smooth

Would you have done things differently in retrospect?

I personally would not do anything differently with Django, besides giving myself more time to learn how to host static files properly. Otherwise, I’m proud of how the website has turned out, and of the demo page’s functionality.

The concept idea for the demo page could have showed off more materialize components.

Did you regret your choice of frameworks?

No. Initially, we were going to build our website using ReactJS as a front end, and NodeJS as a backend. We learned how difficult of a path that was to learn, especially when having little JavaScript experience prior. So, we switched from both of those frameworks to utilizing Materialize as the front end framework and Django as the backend. They were both much simpler to understand, and fit our needs much better.
