Materialize Demo

CSS and Buttons

Materialize CSS is a great framework for making webpages that are visually appealing as it uses the Google design methods that focuses on user experience.
Materialize has a large colour palette to choose from. This allows for a great amount of customizability when creating backgrounds, panels, buttons, or text.

In this tutorial we have five teal buttons with white text. To change the shade of teal simpily type:
lighten/darken-(amount)
Where lighten makes the colour brighter and darken makes it darker.

<a class="btn-large teal lighten-2 white-text btn">Button</a> <a class="btn-large teal lighten-1 white-text btn">Button</a> <a class="btn-large teal white-text btn">Button</a> <a class="btn-large teal darken-1 white-text btn">Button</a> <a class="btn-large teal darken-2 white-text btn">Button</a>

Images

To display images you have the options to choose between cards, material boxes, and parrallax. In this tutorial, we show you how to use parrallax. Parrallax is easy to implement but also needs to be imported throught jQuery.

The Parrallax component <div class="parallax"><img src="https://i.cbc.ca/1.5175344.1560502192!/fileImage/httpImage/image.jpg_gen/derivatives/16x9_780/1155815863.jpg"/></div>

Materialize can set certain CSS styles or hide certain content depending on the device.
To have parrallax work on mobile devices, the height of the image must to be lowered since the image height of parrallax is set to 500 pixels by default.

@media only screen and (max-width: 600px) { .parallax-container { height: 150px; } }

Another method included in this demo is material boxes. These images show when hovered over as it lightens the image. Also, when clicked, the full size of the image will be displayed along with a caption. This must also be initialized through jQuery.

$('.materialboxed').materialbox();

Django Demo

For our demo, we used Django as a backend framework to display Django's model-view-template capabilities.

What is model-view-template?

To begin, a model is a subclass of Django’s Models class that maps each subclass to a database table. It creates the database table and parameters to store the websites information. It’s used to generate dynamic pages easily, as it allows for the automatic creation of databases.

A view is used by Django to tell which files to draw to the screen when called by a url.

A template is used by Django views to display content on the users screen, generally in HTML files. Django uses a special templating engine, called jinja, which adds logical operators to HTML code.

Now let's create a Django website, by exploring templating.

Django’s jinja templating allows developers to avoid unnecessarily repeating code, while maintaining consistent design.
